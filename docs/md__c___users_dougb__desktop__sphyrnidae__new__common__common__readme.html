Sphyrnidae Common Library  2.0.0
Shared Utilities/Library
Overview

This Nuget package contains numerous helper methods and common functionality useful in any application. To install, you can search Nuget for "Sphyrnidae.Common" - and contains no license (is free to all to use). Please note that I do not currently have the source code available for use - when this is available, I'll be sure to update with a proper license. This documentation will guide you through all of the utilities available in this package.

Additionally, I've included reference material for coding standards associated with these packages (although the guidance is generally applicable to any application regardless of this library).

  • API Setup: How to setup your API project
  • API Standards: Best practices for coding an API.

Installation

Internal Nuget Feed Url

To install a nuget package into your solution, you will first need to reference our internal TFS Nuget feed. Current URL is:
https://use1a-tfs.visualvault.com/tfs/auersoft/_packaging/VisualVault/nuget/v3/index.json

To access current URL, go through TFS => Artifacts => Connect to feed => Package source URL


Visual Studio Nuget Feed

Inside visual studio, select Tools => Options => Nuget Package Manager => Package Sources
Name: "VisualVault" (You can call this anything you want, but "VisualVault" makes the most sense)
Source: Paste in the package source URL from above.
Click Update => OK


Install in application

Inside of visual studio, you can right-click on 'Dependencies' inside your project, then select Manage Nuget Packages...
Change the package source to be "VisualVault" (or whatever you named it above).
Under 'Browse' tab, you should see all the packages listed.
You can select any that you'd like/need, and then press the 'Install' button.

If you'd like to install all packages, then you can select the VisualVault.WebApi.Core package.


Packages

Overview

The nuget packages are a collection of capabilities that are common to the entire suite of applications.
Each nuget package generally builds upon the previous package to provide additional capabilities.
Additionally, most packages are built with .net Standard, meaning they can be used in .net Framework, or in .net Core.
The packages that are .net Core specific have a ".Core" extension in the name - and are only available in a .net Core application.

If you're not sure where to start, go here.

Listing

Utilities This is a collection of extension methods, some share models, interfaces, and static wrappers for the interfaces.

Utilities.Core This is additional utility functions that are specific to .net Core.

Logging This is a logging framework similar to other frameworks, but with a great deal of additional capability. Many components in these nuget packages utilize this logging framework.

Logging.Core This is some .net Core specific extension methods to the logging framework.

Dal This is a data access layer (DAL) for connecting to databases, specifically implemented for Sql Server.

Implementations.Common The Implementations namespace is where all of the services are implemented. This package contains generic implementations (non-VisualVault and non-Core). Eg. These could be used by anyone.

Implementations The Implementations namespace is where all of the services are implemented. This package contains VisualVault-specific implementations. Eg. Can not be used outside of the company.

Implementations.Core The Implementations namespace is where all of the services are implemented. This package contains generic implementations that are .net Core-specific implementations. Eg. These could be used by anyone.

WebApi This is common methods that should be used by any Api implementation.

WebApi.Core This provides all of the capabilities of these packages in a single place for consumption by a .net Core application.

SharedEntities This is meant to store any common business logic across multiple applications.

The complete listing with versions and dependencies can be found in the development shared folder => Applications and Settings.xlsx => Nuget tab.
